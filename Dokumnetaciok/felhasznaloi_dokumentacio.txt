Felhasználói Dokumentáció – Autószerviz Nyilvántartó Rendszer

1. A program feladata
Az Autószerviz Nyilvántartó Rendszer egy konzolos alkalmazás, amelynek célja egy kisebb autószerviz mindennapi adminisztrációjának megkönnyítése. A program segítségével a szerviz dolgozói nyilvántarthatják az ügyfeleket, a gépjárműveket és az elvégzett javításokat. Lehetőség van a műszaki vizsgák érvényességének nyomon követésére és a korábbi szerviztörténet lekérdezésére is.

2. A program indítása

Visual Studio:
1. Create new project
2. Console app C++ (C-t is tud futtatni)
3. Jobb egérgombbal kattintunk a projekt nevére, Add -> Existing item, kijelöljük az összes .c és .h fájlt.
4. Jobb egérgombbal kattintunk a projekt nevére, Properties -> C/C++ -> Preprocessor -> Preprocessor Definitions -> bemásoljuk azt, hogy: ;_CRT_SECURE_NO_WARNINGS -> OK
5. Jobb egérgombbal kattintunk a projekt nevére, Open Folder in File Explorer -> bemásoljuk ide a data mappát.
6. Futtatjuk a programot.

Visual Studio Code:
1. GCC telepítése
2. Terminálban: gcc main.c functions.c auto.c ugyfel.c javitas.c -o main
3. Az main.exe fájlra jobb kattintás, majd megnyitjuk (futtatjuk) a main.exe-t.

A program indításkor automatikusan betölti a korábban rögzített adatokat a "data" mappában található szöveges állományokból (ugyfelek.txt, autok.txt, javitasok.txt). Amennyiben ezek a fájlok nem léteznek vagy üresek, a program üres adatbázissal indul.

Fontos: A helyes működéshez biztosítani kell, hogy a futtatható program mellett létezzen egy "data" nevű mappa, és abban a programnak írási joga legyen.

3. A program használata
-----------------------
Indítás után a program egy főmenüt jelenít meg. A navigáció a menüpontok sorszámának (1-9) megadásával, majd az Enter lenyomásával történik.

Főmenü opciók:

1. Ugyfel hozzadasa
2. Auto hozzaadasa
3. Javitas hozzaadasa
4. Kereses ugyfel neve szerint
5. Kereses auto rendszam szerint
6. Auto torlese
7. Auto szerviz tortenet
8. Lejaro vizsgaju autok listazasa
9. Kilepes
Valassz egy menupontot: 

1. Ügyfél hozzáadása
   - Név: Az ügyfél teljes neve (egyedi kell legyen).
   - Email-cím: Az ügyfél email-címe.
   - Telefonszám: Az ügyfél telefonszáma.
   - Mentés megerősítése: 'I' (Igen) betűvel vagy megszkítható 'N' (Nem) betűvel

2. Autó hozzáadása
   Új gépjármű felvétele.
   - Rendszám: Egyedi azonosító (pl. ABC-123).
   - Modell: Az autó típusa (pl. Opel Astra).
   - Vizsga érvényessége: ÉÉÉÉ-HH-NN formátumban (pl. 2025-10-31).
   - Tulajdonos neve: Összekapcsolja az autót a már rögzített ügyféllel.
   - Mentés megerősítése: 'I' (Igen) betűvel vagy megszkítható 'N' (Nem) betűvel

3. Javítás hozzáadása
   Szervizmunkálat rögzítése meglévő autóhoz.
   - Autó rendszáma: Melyik autóhoz tartozik.
   - Javítás típusa: Munka leírása (pl. Olajcsere).
   - Dátum: A javítás napja (ÉÉÉÉ-HH-NN).
   - Ár: Költség forintban.
   - Mentés megerősítése: 'I' (Igen) betűvel vagy megszkítható 'N' (Nem) betűvel

4. Keresés ügyfél neve szerint
   Név vagy névrészlet alapján listázza az ügyfél adatait (név, email, telefon).

5. Keresés autó rendszám szerint
   Rendszám alapján megjeleníti az autó modelljét, vizsgaérvényességét és a tulajdonos nevét.

6. Autó törlése
   Gépjármű eltávolítása rendszám alapján.
   - Sikeres találat esetén a program rákérdez a kapcsolódó javítások törlésére is (I/N). Javasolt az 'I' választása a rendszer tisztasága érdekében.

7. Autó szerviz történet
   Adott rendszámú autó összes korábbi javításának listázása (dátum, típus, ár).

8. Lejáró vizsgájú autók listázása
   Kiszűri azokat az autókat, amelyek műszaki vizsgája egy megadott dátumhoz képest lejárt vagy érvénytelen.
   - A felhasználónak meg kell adnia a viszonyítási dátumot (pl. a mai napot) ÉÉÉÉ-HH-NN formátumban.

9. Kilépés
   A program bezárása és az adatok mentése.
   Fontos: A változtatások csak a szabályos kilépés során kerülnek mentésre a fájlokba!

4. Adattárolás
--------------
Az adatok a "data/" könyvtárban, pontosvesszővel (;) tagolt szöveges fájlokban tárolódnak:
- ugyfelek.txt: Név;Email;Telefonszám
- autok.txt: Rendszám;Modell;VizsgaÉrvényesség;TulajdonosNeve
- javitasok.txt: Rendszám;JavításTípusa;Dátum;Ár

5. Hibalehetőségek és megoldások
--------------------------------
- Hibás dátumformátum: Ügyeljen az ÉÉÉÉ-HH-NN formátum betartására (kötőjelekkel).
- Már létező adat: Nem vehető fel két azonos nevű ügyfél vagy azonos rendszámú autó.
- Hiányzó mappa: Ha a "data" mappa hiányzik, a program nem tud menteni. Indítás előtt hozza létre a mappát.
